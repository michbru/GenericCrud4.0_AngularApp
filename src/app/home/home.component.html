4.0 Hello Generic CRUD .NET Core EF with Dapper and Angular !!

Welcome to your new single-page application, built with:

  • ASP.NET Core and C# for cross-platform server-side code
  • Angular (Angular Cli) and TypeScript for client-side code
  • Bootstrap for layout and styling

This is my 4th Iteration on this project. Below in descending order are high points of evolution:

  • 4.0 Generic CRUD Here I have removed (for now) nuget package dependency and in so doing replaced some of the .NET Reflection code with the Dapper dependency. Technology Elements of current iteration include:
    • .NET Core 3.x
    • Swashbuckle/Swagger API Page
    • Entity Framework and Dapper for Data Access/Management
    • Single Generic Controller and Generic Service for CRUD
    • SQL Server Database
    • Angular 8 UI with Generic Component for all CRUD
  • 3.x Generic CRUD Here I was still using .NET Framework (no Core) and No Dapper Technology Elements of that iteration included:
    • .NET 4.x with Entity Framework, SQL Server and Angular
    • Custom Nuget Package containing Reflection code for the Generic CRUD
    • Single Generic Controller and Generic Service for CRUD
  • 2.x Generic CRUD .NET Framework with Angular.js Technology Elements of that iteration included:
    • .NET with Entity Framework, SQL Server and Angular.js
    • Single Generic Controller and Generic Service for CRUD
  • 1.x Generic CRUD Active Server Pages Technology Elements of that iteration included:
    • ASP.NET, ADO with SQL Server
    • Single Generic Controller and Generic Service for CRUD
